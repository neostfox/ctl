# Quality Guidelines

> Code quality standards for the AI Dev Control Plane project.

---

## Overview

This project enforces strict architectural boundaries (see `ARCHITECTURE_GUARDRAILS.md`). Code quality means **boundary compliance**, **deterministic behavior**, and **testability** — not just passing lints.

---

## Forbidden Patterns

### F1: Side Effects in `domain/`

```rust
// FORBIDDEN — domain must be pure
use std::fs;
let content = fs::read_to_string("events.jsonl")?;
let now = std::time::SystemTime::now();
std::process::Command::new("cargo").args(&["test"]).output()?;
```

**Why**: STATE-001, MODULE-002. The reducer (`apply`) must be replayable without environment.

### F2: Direct `task.json` Writes

```rust
// FORBIDDEN — task.json is a projection
fs::write("task.json", manually_constructed_json)?;
```

**Why**: STATE-002. `task.json` is only generated by `replay`. Hand-writing it breaks the truth model.

### F3: In-Place `events.jsonl` Mutation

```rust
// FORBIDDEN — events.jsonl is append-only
let content = fs::read_to_string("events.jsonl")?;
let modified = content.replace(old_event, new_event);
fs::write("events.jsonl", modified)?;
```

**Why**: STATE-001. Canonical events are immutable once appended.

### F4: Legacy `scope` Field

```rust
// FORBIDDEN — use read_scope + write_allow + write_deny
let payload = json!({ "scope": ["src/"] });
```

**Why**: `scope` cannot express read vs write permission. Must use structured boundary fields.

### F5: Arbitrary Shell Execution

```rust
// FORBIDDEN — only predefined gate templates
std::process::Command::new(user_provided_command).output()?;
```

**Why**: EXEC-001. M0–M3 only allows predefined `cargo_*` gate templates.

### F6: Async Runtime

```rust
// FORBIDDEN before M3
async fn do_something() { /* ... */ }
#[tokio::main]
async fn main() { /* ... */ }
```

**Why**: DEP-002. No async runtime, HTTP client, or database before M3.

### F7: Unvalidated Path in Boundary Decision

```rust
// FORBIDDEN — must normalize first
if write_allow.contains(&user_input_path) { /* ... */ }

// REQUIRED — normalize then check
let normalized = normalizer.normalize(&user_input_path)?;
```

**Why**: PATH-001. Un-normalized paths bypass boundary checks.

## Required Patterns

### R1: Validate → Build → Append → Rebuild

Every command follows this sequence:

```rust
// 1. Validate inputs
validate_task_definition(objective, &read_scope, &write_allow, &gates)?;
// 2. Build event
let event = self.build_event(task_id, "task_created", payload)?;
// 3. Schema-validate + append
self.validate_and_append(&event)?;
// 4. Rebuild projection
self.rebuild_task_view(task_id)?;
```

### R2: Replay for Current State

Always replay from `events.jsonl` to get current state — never cache stale `TaskState`:

```rust
let state = self.replay_task(task_id)?;
```

### R3: Atomic Projection Writes

Write projections via temp file + rename:

```rust
let temp_path = task_dir.join("task.json.tmp");
fs::write(&temp_path, content)?;
fs::rename(&temp_path, task_dir.join("task.json"))?;
```

### R4: Deterministic Event Building

Events use strict ascending `seq` and unique `command_id`:

```rust
static COUNTER: AtomicU64 = AtomicU64::new(1);
let seq = COUNTER.fetch_add(1, Ordering::Relaxed);
let command_id = format!("cmd-{}-{}", task_id, seq);
```

### R5: Gate Template Registry

All gate IDs must be registered in `infrastructure/gates/mod.rs`:

```rust
pub static GATE_TEMPLATES: &[GateTemplate] = &[
    GateTemplate { id: "cargo_check", ... },
    GateTemplate { id: "cargo_test", ... },
    // ...
];
```

Unknown gate IDs are rejected at validation time.

## Testing Requirements

### Unit Tests

- **Reducer**: Every `apply` branch must have at least one test. Use `fixtures/*.jsonl` for lifecycle coverage.
- **Path Normalizer**: Test every rejection case (absolute, `..`, UNC, symlink, junction, root-escape, protected paths).
- **Schema Validator**: Valid and invalid envelopes for each schema.

### Integration Tests

- **Store Round-Trip**: Append event → read back → verify byte-identical.
- **Lifecycle**: Create → Ready → Start → Submit → Finish in temp workspace.
- **Replay Determinism**: Same events → same `task.json` bytes.

### Architecture Tests

- **Command Surface**: `control architecture check` verifies M1-only commands.
- **Dependency Direction**: No `domain/` imports from `cli/`, `infrastructure/`, `adapters/`.
- **Fixture Compliance**: Fixtures use canonical gate IDs and structured boundaries.

### Test Location

- Unit tests: inline `#[cfg(test)] mod tests` in each module.
- Integration tests: `tests/audit_matrix.rs` and `main.rs` test module.
- Fixtures: `fixtures/*.jsonl` for canonical event sequences.

## Code Review Checklist

- [ ] Change is scoped to current milestone (M0–M3)
- [ ] No new runtime dependency without DEP-001 review
- [ ] `domain/` remains pure (no I/O imports)
- [ ] New event types have reducer branches + fixture coverage
- [ ] Path inputs go through `PathNormalizer`
- [ ] Projections use atomic temp-file write
- [ ] Error messages include identifiers and suggest next actions
- [ ] `cargo clippy -- -D warnings` clean
